# Логика проверки: "Я НЕ админ" - что происходит?

## Полный процесс проверки роли:

### ШАГ 1: Пользователь вводит email и password
```
Форма отправляется на: POST /login
Данные: email, password
```

### ШАГ 2: Валидация данных (AdminLoginController@login)
```php
$credentials = $request->validate([
    'email' => ['required', 'email'],    // Проверка: email обязателен и валидный
    'password' => ['required'],           // Проверка: пароль обязателен
]);
```
**Результат:**
- ✅ Если валидация прошла → продолжаем
- ❌ Если нет → возврат на страницу с ошибками

### ШАГ 3: Попытка аутентификации
```php
if (Auth::attempt($credentials)) {
    // Что делает Auth::attempt():
    // 1. Ищет пользователя в БД по email
    // 2. Проверяет пароль (сравнивает хеш)
    // 3. Если совпадает → создает сессию, возвращает TRUE
    // 4. Если нет → возвращает FALSE
}
```

**Сценарий А: Неправильный email/password**
```
Auth::attempt() → FALSE
    ↓
return back()->withErrors(['email' => 'Неверный логин или пароль.']);
    ↓
Пользователь видит ошибку на странице логина
```

**Сценарий Б: Правильный email/password, НО роль НЕ 'admin'**
```
Auth::attempt() → TRUE ✅
    ↓
$request->session()->regenerate();  // Обновляет сессию
    ↓
$user = Auth::user();  // Получает объект пользователя из БД
    ↓
Проверка роли:
if ($user && ($user->role ?? null) === 'admin') {
    // Это НЕ выполнится, если role != 'admin'
}
    ↓
Переход в блок "Если зашел не админ":
Auth::logout();  // ❌ ВЫЛОГИНИВАЕМ пользователя
    ↓
return back()->withErrors(['email' => 'Доступ только для администраторов.']);
    ↓
Пользователь видит ошибку: "Доступ только для администраторов"
```

### ШАГ 4: Что происходит в базе данных?

**Таблица users:**
```
id | name      | email              | password (хеш) | role  |
1  | Admin     | admin@example.com  | $2y$10$...     | admin | ✅
2  | User      | user@example.com   | $2y$10$...     | user  | ❌
```

**Процесс:**
1. `Auth::attempt()` находит пользователя по email
2. Проверяет пароль (сравнивает хеш)
3. Если пароль правильный → загружает ВСЕ данные пользователя, включая `role`
4. Сохраняет пользователя в сессию
5. Контроллер проверяет: `$user->role === 'admin'`
6. Если `role = 'user'` или `null` → НЕ админ → logout

### ШАГ 5: Что происходит после logout?

```php
Auth::logout();
```
**Это делает:**
- Удаляет пользователя из сессии
- Очищает данные авторизации
- Пользователь становится "незалогиненным"

```php
return back()->withErrors(['email' => 'Доступ только для администраторов.']);
```
**Это делает:**
- Возвращает на страницу `/login`
- Передает ошибку в сессию
- View отображает ошибку: "Доступ только для администраторов"

## Визуальная схема:

```
Пользователь вводит данные
    ↓
Валидация ✅
    ↓
Auth::attempt() 
    ↓
    ├─→ FALSE (неправильный пароль)
    │       ↓
    │   Ошибка: "Неверный логин или пароль"
    │
    └─→ TRUE (правильный пароль)
            ↓
        Загружается пользователь из БД
            ↓
        Проверка: $user->role === 'admin'?
            ↓
            ├─→ ДА ✅
            │       ↓
            │   redirect('/admin')
            │   Пользователь видит админку
            │
            └─→ НЕТ ❌
                    ↓
                Auth::logout()
                    ↓
                Ошибка: "Доступ только для администраторов"
                    ↓
                Пользователь остается на /login
                Сессия очищена
```

## Важные моменты:

1. **Пользователь УСПЕШНО авторизовался** (пароль правильный)
2. **НО его роль НЕ 'admin'**
3. **Система ВЫЛОГИНИВАЕТ его** (защита от несанкционированного доступа)
4. **Показывает ошибку** вместо админки

Это двухуровневая защита:
- Уровень 1: Правильный пароль (Auth::attempt)
- Уровень 2: Правильная роль (role === 'admin')
